# AIProxyHub 使用指南

## 一、快速开始（一键模式）

> **EXE 版说明（下载即用）**：如果你拿到的是 `AIProxyHub-*-win64.zip` 发布包，解压后直接双击运行 `AIProxyHub.exe` 即可（无需 Python / 无需 `启动.bat`）。  
> **数据保存位置（重要）**：EXE 版默认把配置与 `data/` 写入 **`%LOCALAPPDATA%\\AIProxyHub`**（更符合“安装版软件”的习惯，可避免升级/卸载误删数据）。  
> 如需“便携模式（数据与 EXE 同目录）”，请在 `AIProxyHub.exe` 同目录创建一个空文件：`AIProxyHub.portable`。也可用环境变量 `AIPROXYHUB_HOME` 强制指定数据目录。

### 1. 首次配置

双击 `启动.bat`，浏览器自动打开管理面板。

> 新版启动脚本会自动创建/复用本地虚拟环境 `.venv` 并安装依赖（例如 `curl_cffi`），避免“换机器就跑不起来”的问题。

进入 **配置** 页面，填写以下必要信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| DuckMail API Token | 从 [duckmail.sbs](https://duckmail.sbs) 注册获取 | `dk_xxxxxxxx...` |
| 代理地址 | 你的科学上网代理（必须可用） | `http://127.0.0.1:7890` |
| 注册数量 | 每次注册的账号数 | `5` |
| 并发数 | 同时注册的线程数（建议 3） | `3` |
| 管理密码 | CPA 管理面板密码 | `<自动生成>` |
| API Key | 客户端连接代理时使用的密钥 | `<自动生成>` |

点击 **保存配置**。

> 出于安全考虑：保存后 DuckMail Token / 管理密码 / API Key 默认不会回显；如需修改请重新输入，留空表示“保持不变”。首次启动如果没有现成配置，系统会自动生成强随机「管理密码 / API Key」，可在配置页点击“显示/复制”获取。
>
> 另外：默认更安全 —— 不会把 ChatGPT/邮箱密码写入 `registered_accounts.txt`，也不会生成 `ak.txt / rk.txt` 明文 token 文件。如需调试/落盘保存，可在「配置 → 安全输出（高级）」中显式开启。

### 2. 一键全流程

在 **仪表盘** 页面，点击顶部的 **「一键启动」** 按钮。

如果流程卡住或想中止，可使用页面中的 **「停止注册 / 停止一键流程」** 按钮。

系统自动执行：
1. 启动 CLIProxyAPI 代理服务
2. 清理过期/无效账号
3. 批量注册新 ChatGPT 账号
4. 自动获取 OAuth Token 并上传到代理

进度条实时显示当前阶段，全部完成后显示接入信息。

### 3. 使用 API

注册完成后，在任意兼容 OpenAI 的客户端中配置：

```
API Base URL: http://localhost:8317/v1
API Key: <api_key>（你在配置页设置/复制的密钥）
```

支持的客户端：Cursor、Claude Code、Cline、RooCode、Continue、OpenCode 等。

## 二、注册技巧

### DuckMail Token 获取

1. 访问 [duckmail.sbs](https://duckmail.sbs)
2. 注册账号，在控制台获取 API Bearer Token
3. Token 格式为 `dk_` 开头的字符串

### 代理要求

- **必须使用代理**，否则 IP 会被 OpenAI 封禁
- 推荐使用美国/欧洲节点
- 代理格式：`http://host:port` 或 `socks5://host:port`
- 确保代理稳定，注册过程中断线会导致失败

### 注册数量建议

| 场景 | 建议数量 | 并发数 |
|------|---------|--------|
| 个人使用 | 3-5 | 2-3 |
| 小团队 | 10-20 | 3-5 |
| 高频使用 | 20+ | 5-8 |

- 并发数不要超过 8，否则可能触发 OpenAI 频率限制
- 每个账号有独立配额，账号越多可用额度越大
- 代理会自动轮询所有账号，均匀分配请求

### 注册失败排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 无任何输出 | 代理不通 | 检查代理是否可用 |
| DuckMail 报错 | Token 过期 | 重新获取 Token |
| OAuth 失败 | IP 被封 | 更换代理节点 |
| 上传失败 | 代理未启动 | 先启动代理再注册 |

## 三、账号管理

### 查看账号

在 **账号管理** 页面可以看到所有已注册账号，包括邮箱、类型、注册时间。

### 清理无效账号

点击 **「清理无效账号」** 按钮，系统自动检测并删除：
- `disabled: true` 的账号
- 已过期的账号（`expired` 早于当前时间）
- 缺少 `access_token` 和 `refresh_token` 的账号
- JSON 文件损坏的账号

建议在每次注册前执行一次清理。

### 手动删除

点击账号行右侧的 **删除** 按钮可以删除单个账号。

### 安全清理敏感输出（推荐）

在 **账号管理** 页面「输出文件」区域点击 **「安全清理敏感输出」**，会：
- 将 `registered_accounts.txt` 中的密码字段覆盖为 `***`（保留账号列表）
- 删除 `data/ak.txt`、`data/rk.txt`（明文 token）
- 清理 `codex_tokens/*.json`（token_json 调试产物）
- 删除 `%TEMP%\\AIProxyHub\\*.runtime.*`（运行时临时配置，异常退出可能残留；若相关进程仍在运行则跳过）
- 删除 `%TEMP%\\AIProxyHub\\_test_register_cfg.json`（历史/测试残留，可能包含 DuckMail Token）

> 不会影响代理服务实际使用的账号文件（`~/.cli-proxy-api/`）。

## 四、代理服务配置

### 路由策略

| 策略 | 说明 |
|------|------|
| `round-robin` | 轮询（默认），请求均匀分配到所有账号 |
| `fill-first` | 填满优先，优先使用第一个账号直到配额耗尽 |

### 配额超限自动切换

开启后，当一个账号配额用完，代理会自动切换到下一个账号，无需手动干预。

### 缓存池（跨账号节省额度）

> 适用场景：同一批提示词/任务会被重复调用（例如团队共享 prompt、自动化脚本重试、多人并发问同一个问题）。

在 **配置 → 缓存池（跨账号节省额度）** 中：

1) 勾选 **「启用透明网关（不改 Base URL）」**（推荐）  
2) 勾选 **「启用响应缓存（仅非 stream）」**  
3) （可选）勾选 **「启用 stream 缓存（SSE/WS，命中时快速回放）」**（更适合重复任务/压测；命中时延迟更低）  
4) （可选）勾选 **「跨 API Key 共享缓存（仅可信环境）」**（团队多人共用同一套 prompt 时可提高命中率）  
5) 配置缓存参数：
   - 缓存 TTL（秒）：默认 3600（1 小时）
   - TTL 抖动（秒）：默认 30（建议 10~60，避免大量条目同一时刻过期造成抖动）
   - 总内存上限（MB）：默认 128（建议 64~512，避免高并发下缓存把内存打爆）
   - 最大条目数：默认 200（LRU）
   - 单条最大大小（KB）：默认 512（避免大响应占满内存）
   - SWR（stale-while-revalidate，秒）：默认 0（关闭）
     - 过期后在该窗口内仍可返回旧缓存，同时后台刷新（后台刷新会产生真实回源请求，可能消耗额度）
   - SIE（stale-if-error，秒）：默认 0（关闭）
     - 过期后在该窗口内若回源失败/限流，可返回旧缓存兜底（适合“稳定优先”的场景）
   - Vary Headers（逗号分隔）：默认包含 `OpenAI-Project,OpenAI-Organization,X-AIProxyHub-Cache-Group`
     - 用于把这些 header 纳入缓存 key，避免不同项目/租户上下文误命中同一缓存
6) 点击 **保存配置**，若页面提示需要重启：到 **仪表盘** 点击 **「重启代理」** 应用配置。

缓存覆盖范围（当前版本）：
- `POST /v1/responses`（`stream=false`）
- `POST /v1/chat/completions`（`stream=false`）
- （开启「启用 stream 缓存」时）`POST /v1/responses`（`stream=true`，SSE 事件流“快速回放”）
- （开启「启用 stream 缓存」时）`/v1/responses` WebSocket mode（事件流“快速回放”）

注意事项：
- 缓存会把请求与响应暂存在内存中（不落盘），**命中缓存不会消耗任何账号额度**。
- 如果你的请求包含敏感信息（如隐私文本/业务机密），不建议开启缓存，或将 TTL 设置得很短。
- `stream=true` 的流式响应 **默认不缓存**（直接透传）；仅当你勾选了「启用 stream 缓存」时，才会捕获并缓存事件流（命中时快速回放）。
- 默认按客户端 API Key 隔离缓存；开启「跨 API Key 共享缓存」后，不同调用方会共享同一份缓存结果（仅适用于可信环境）。
- 开启「跨 API Key 共享缓存」并不会关闭鉴权：仍需要携带有效 API Key；只是缓存 key 不再包含 Authorization，从而提升多人共享场景命中率。
- 请求级控制（可选）：
  - `X-AIProxyHub-Cache: bypass`：绕过读取 + 不写入（强制回源，不污染缓存）
  - `X-AIProxyHub-Cache: refresh`：绕过读取但允许写入（刷新缓存）
  - `X-AIProxyHub-Cache: no-store`：不写入但允许读（如已有缓存则可命中）

### 高并发（例如 20 req/s）怎么保证？

要稳定达到 20 req/s，关键不是“单机能跑”，而是**让每次请求尽量少打到真实账号/上游**：

1) **强烈建议开启缓存池**（尤其是团队重复调用同一批 prompt 的场景）  
   - 同一请求命中缓存时，网关直接返回结果，延迟更低且不消耗额度；20 req/s 级别更容易达成。

2) **请求是否“高度重复”决定你的上限**  
   - 高度重复（命中缓存）：主要瓶颈在本机网关吞吐与本地网络，可轻松到 20 req/s 甚至更高。  
   - 高度不重复（基本不命中缓存）：瓶颈在账号数量/额度/上游速率限制，此时需要更多账号（或降低并发/做排队）。

3) **团队共享缓存时，建议使用分组头隔离任务**  
   - 客户端可选传递：`X-AIProxyHub-Cache-Group: <team-or-job-id>`  
   - 好处：同一台机器上不同团队/不同任务不会“误命中”彼此缓存（更安全、更可控）。

### CPA 管理面板

点击仪表盘的 **「CPA 管理面板」** 按钮，可以打开 CLIProxyAPI 自带的管理界面，查看更详细的账号状态和请求日志。

访问地址：`http://localhost:8317/management.html`

> 重要：首次打开会进入登录页，需要输入 **管理密钥**（即你在 AIProxyHub「配置」页看到的 **管理密码**）。  
> 这和客户端调用 `/v1/*` 时使用的 **API Key** 不是同一个东西：  
> - **管理密码**：用于登录 CPA 管理中心（`/v0/management/*`）  
> - **API Key**：用于客户端调用代理（`/v1/models`、`/v1/responses` 等）

#### 为什么“使用统计”没显示？

常见原因与处理：
1) **未使用管理密码登录**：未登录时无法读取 `/v0/management/usage`，页面会显示为空/报鉴权失败。  
2) **没有真实请求**：使用统计只有在你通过代理发起过 `/v1/responses`（或其它 `/v1/*` 推理请求）后才会增长；仅打开页面不会产生统计。  
3) **开关未开启**：需要 `usage-statistics-enabled: true`。本项目新版会在运行时自动写入该开关（你在 CPA 仪表盘也能看到“启用使用统计=是”）。
4) **开启了缓存池**：缓存命中不会打到代理后端，因此 **CPA usage 不会增长**（这是节省额度的正常表现）。可在 AIProxyHub 仪表盘查看「缓存命中/命中率」，以及「总请求数/总 Tokens」（来自 CPA 汇总）。

## 五、文件说明

```
E:\AIProxyHub\
├── 启动.bat                 # 双击启动
├── launcher.py              # 管理后端 + Web UI
├── settings.json            # 统一配置文件
├── config.yaml              # （历史版本遗留）CLIProxyAPI 配置文件（新版默认不再在项目目录生成）
├── cli-proxy-api.exe        # 代理服务
├── register/
│   ├── chatgpt_register.py  # 注册脚本
│   └── codex/
│       └── protocol_keygen.py  # 高级/实验脚本（默认流程不使用）
└── data/                    # 注册输出
    ├── registered_accounts.txt    # 默认不含密码（如开启“保存密码”则会包含）
    ├── ak.txt                     # 默认不生成（如开启“明文输出”则生成）
    └── rk.txt                     # 默认不生成（如开启“明文输出”则生成）
```

### 运行时临时配置（新版）

为降低明文配置长期落盘风险，新版默认将运行时配置写入系统临时目录并在进程退出后清理：

- `%TEMP%\\AIProxyHub\\cli-proxy-api.runtime.yaml`（代理运行时配置）
- `%TEMP%\\AIProxyHub\\register.runtime.json`（注册运行时配置）
- `%TEMP%\\AIProxyHub\\codex_tokens\\*.json`（token_json 临时文件：默认上传成功后自动清理；如开启“保留”则写入 `data\\codex_tokens\\`）

### 数据目录

- `~/.cli-proxy-api/` — 账号认证文件（JSON），代理服务读取此目录
- `data/` — 注册结果文本文件

> 面板中展示的输出文件默认会做脱敏。默认配置下磁盘文件也不会包含密码/token 明文；如果你在「安全输出（高级）」中开启了落盘保存，则磁盘文件会包含敏感信息，请自行妥善保管并按需清理。

## 六、常见问题

**Q: 注册的账号能用多久？**
A: 账号文件中有 `expired` 字段（通常 10 天），过期后代理会自动刷新 Token。如果刷新失败，清理后重新注册即可。

**Q: 可以在其他电脑上使用代理吗？**
A: 默认只监听 `127.0.0.1`（本地）。如需局域网访问，在配置中将「监听地址」改为 `0.0.0.0`，并确保「管理密码 / API Key」为强随机值；系统在非本地监听时会对弱口令/弱 key 进行拦截，避免被局域网访问。

**Q: 如何增加更多账号？**
A: 在配置中修改注册数量，再次点击「一键启动」或手动点击「开始注册」。新账号会自动追加到现有账号池中。

**Q: 点击「启动代理」会弹出黑色命令行窗口，然后又立刻消失？**
A: 这通常是两类原因之一：  
1) **重复打开了多个 AIProxyHub 实例**：多个面板同时操作同一端口（例如 8317），会导致 CLIProxyAPI 启动后因端口占用立刻退出，从而出现“黑框闪一下就没了”。  
   - 处理：先把多余的 AIProxyHub 全部关闭，只保留一个实例，再启动代理。  
2) **代理进程启动即退出**：例如 `cli-proxy-api.exe` 不存在、配置文件异常、端口被其它程序占用。  
   - 处理：在 AIProxyHub 面板的「运行日志」中查看 `[PROXY]` 输出，会有明确的退出原因。  

> 提示：从 v1.2.7 起，Windows 安装版默认启用“单实例”并隐藏 CLIProxyAPI 的控制台窗口，可显著降低该类误判与闪框干扰。  
> 如果仪表盘显示 **「运行中（外部）」**，表示代理端口上已有其它实例/旧版本启动的代理在运行；此时 AIProxyHub 会避免重复启动以防冲突，但你可能无法在当前面板里停止/重启它（建议关闭其它实例或更换端口）。

**Q: 代理支持哪些模型？**
A: 代理支持 OpenAI Codex 系列模型（GPT-5、GPT-5-Codex 等），具体可用模型取决于注册账号的权限。
